دست راست — فاز ۳ (UI + داشبورد + دستیار)
=========================================

اجرا:
  npm install
  npm start
  → http://localhost:8787
  ادمین پیش‌فرض: admin@dastrast.local / Admin12345

۱) رفع چیدمان منوهای چسبان
   - body حالا fixed/inset-0 و overflow:hidden است؛ کل صفحه دیگر اسکرول
     نمی‌شود و فقط محتوای داخل اپ اسکرول می‌شود.
   - main از h-full max-h-[100dvh] استفاده می‌کند؛ فضای خالی پایین و
     جابجا شدن منوی بالا/پایین در بخش دستیار از بین رفت.
   - overscroll-behavior:none برای جلوگیری از کشیده‌شدن صفحه.

۲) موج صوتی نرم و متحرک هنگام ضبط ویس (مثل Siri / Gemini)
   - کامپوننت VoiceWave روی <canvas> سه لایه موج نرم رسم می‌کند.
   - اگر دسترسی میکروفون داده شود، دامنهٔ موج از روی فرکانس واقعی صدا
     (Web Audio API / AnalyserNode) محاسبه می‌شود.
   - اگر دسترسی نبود، موج آرام شبیه‌سازی‌شده اجرا می‌شود (fallback).
   - دایرهٔ میکروفون با هالهٔ نرم (mic-glow) نبض می‌زند.
   - تشخیص گفتار interimResults فعال شد تا متن همان لحظه دیده شود.

۳) تکمیل داشبورد مالی (کارکرد واقعی)
   - نمودار میله‌ای روند درآمد/هزینه ۱۲ ماه اخیر (SVG درون‌خطی).
   - نمودار دونات سهم دسته‌بندی هزینه‌ها.
   - بازهٔ زمانی: ماهانه / فصلی / سالانه / کل (فیلتر واقعی سمت سرور).
   - مقایسهٔ دوره‌ای: درآمد/هزینه/مانده نسبت به دورهٔ قبل با درصد تغییر.
   - هشدارهای مالی هوشمند: هزینه بیش از درآمد، جهش هزینه، چک نزدیک
     سررسید، بزرگ‌ترین بدهکار/بستانکار.
   - فیلتر بر اساس دسته‌بندی.
   Endpointهای جدید سرور:
     GET /api/analytics/summary?range=&category=&personId=&type=
     GET /api/analytics/compare
     GET /api/analytics/alerts

۴) تقویت دستیار هوشمند
   - رفع باگ تشخیص نام: «از علی قرض گرفتم» → علی (نه «علی قرض»).
     واژه‌های ایست (پول، قرض، دادم...) از انتهای نام بریده می‌شوند.
   - رفع ابهام مرحله‌ای با دکمه‌های شیشه‌ای واقعی: اگر چند «علی» باشد،
     سوال + دکمه برای انتخاب دقیق نمایش داده می‌شود و دستور دوباره با
     نام کامل اجرا می‌شود.
   - ویرایش آخرین تراکنش با دستور طبیعی:
     «مبلغ آخرین تراکنش رو ۹۰ تومن کن» / «عنوان آخری رو ... کن».
   - تشخیص چند عملیات بدون جداکننده:
     «۳۰ تومن تاکسی دادم و ۵۰ تومن ناهار خریدم» → دو تراکنش جداگانه.
   - این بهبودها هم در server/index.js و هم در server-standalone.js اعمال شد.

تست‌های انجام‌شده (همه سبز)
  - person extraction → «علی»
  - multi-op بدون جداکننده → ۲ عملیات
  - edit → action=edited
  - clarification → needsClarification + candidates
  - analytics/summary, compare, alerts → 200
  - GET / → 200 و عنوان فارسی

نکته بازسازی index.html
  فایل مبدأ index.html قبلاً از مخزن حذف شده بود و فقط dist/index.html
  (بیلدشده) مانده بود؛ به همین دلیل «npm run build» شکست می‌خورد.
  index.html مبدأ بازسازی شد و حالا build سالم است.
