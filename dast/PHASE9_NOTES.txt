دست راست — فاز ۹ (یکپارچگی کامل منطق حسابداری + UX داخلی)
==========================================================

اجرا: npm install && npm start → http://localhost:8787
ادمین: admin@dastrast.local / Admin12345

== ۱) رنگ متن‌ها در حالت روشن + پاپ‌آپ‌های داخلی ==
- دکمه‌های شیشه‌ای با کلاس glass-btn (کنتراست درست در روشن و تاریک).
- همهٔ تاییدها/ورودی‌ها از پاپ‌آپ داخلی استفاده می‌کنند (هیچ پنجرهٔ
  مرورگری باقی نمانده — confirm/alert/prompt حذف شد).

== ۲) منوی لمسی + انتخاب گروهی ==
- کامپوننت ActionRow + هوک useLongPress: کلیک‌راست (دسکتاپ) و لمس طولانی
  (موبایل) → منوی داخلی عملیات. روی تراکنش‌ها و چک‌ها فعال است.
- حالت انتخاب گروهی روی «تاریخچه تراکنش‌ها» با حذف دسته‌جمعی و جمع مبلغ.

== ۳) عملیات در دفتر حساب اشخاص ==
- داخل صفحهٔ هر شخص، روی هر تراکنش دفتر حساب: ویرایش و حذف اضافه شد
  (با بازمحاسبهٔ مانده و سند).

== ۴) بازنویسی کامل و یکپارچهٔ منطق حسابداری ==
تک‌منبع حقیقت = «سند حسابداری دوطرفه». همهٔ مانده‌ها از روی دفتر اسناد
محاسبه می‌شوند (نه ذخیرهٔ جداگانه). اصلاحات کلیدی:
- فروش/خرید نسیه (طلب/بدهی) دیگر اشتباهاً وارد «نقد» نمی‌شود؛ به
  «حساب‌های دریافتنی/پرداختنی» می‌رود.
- مانده هر حساب بانک/صندوق = مانده اولیه + (بدهکار−بستانکار) همان حساب
  در دفتر اسناد.
- پاس‌کردن چک، واریز/برداشت، انتقال، تراکنش و تسویه همگی فقط یک سند
  متوازن می‌سازند (پایان دوباره‌شماری).
- ویرایش/حذف هر تراکنش/گردش/چک، سند مرتبط را بازمی‌سازد (refId) تا دفتر
  همیشه هماهنگ بماند.
- انتقال داخلی بین دو حساب در جریان نقد و سود/زیان خنثی است.

تست یکپارچگی (هر دو سرور یکسان):
 فروش نقدی ۱۰ + طلب نسیه ۵ + هزینه ۳ + واریز ۲ + انتقال ۱ →
 صندوق=۶، بانک=۳، علی=+۵ (طلب، نه نقد)، تراز متوازن،
 سود خالص=۱۲، جریان نقد خالص=۹ (انتقال خنثی).

== ۵) صورت سود و زیان و جریان نقدی (کامل) ==
سود و زیان: تفکیک عملیاتی/غیرعملیاتی، بهای تمام‌شده، سود ناخالص،
حاشیهٔ سود، بازهٔ زمانی، نمودار دونات/میله.
جریان نقدی: تفکیک عملیاتی/سرمایه‌گذاری/تامین مالی از روی دفتر اسناد،
بازهٔ زمانی، ردیف‌های دسته‌بندی‌شده. انتقال داخلی خنثی.

== تقویم شمسی ==
JalaliDatePicker با رنگ برند در فیلدهای تاریخ چک و سند.

== دستیار ==
همهٔ عملیات و گزارش‌ها (سود/زیان، تراز، چک، خزانه، حساب‌ها) با همان
موتور حسابداری یکپارچه کار می‌کنند.

همه در server/index.js و server-standalone.js هماهنگ و تست‌شده.
