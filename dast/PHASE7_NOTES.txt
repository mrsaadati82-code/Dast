دست راست — فاز ۷ (ویرایش/حذف همه‌جا + چک + خزانه + دفتر کل)
============================================================

اجرا: npm install && npm start → http://localhost:8787
یا RUN-WINDOWS-NO-INSTALL.bat | ادمین: admin@dastrast.local / Admin12345

== ۱) ویرایش و حذف در همهٔ بخش‌ها (UI + دستیار) ==
- تراکنش‌ها: ویرایش با پاپ‌آپ + حذف در «تاریخچه» و «درآمد/هزینه».
- چک‌ها، حساب‌ها، سرفصل‌ها، اسناد، گردش خزانه، اشخاص، پروژه‌ها، فاکتورها:
  همگی ویرایش/حذف از رابط کاربری دارند.
- دستیار: ویرایش/حذف آخرین تراکنش و چک، پاس‌کردن چک، ویرایش چک،
  حذف با تایید دو مرحله‌ای و بازگردانی (Undo).

== ۲) فیلتر و مرتب‌سازی ==
- چک‌ها: فیلتر نوع/وضعیت + مرتب‌سازی سررسید/مبلغ.
- تاریخچه: فیلتر نوع/دسته + ۴ حالت مرتب‌سازی.
- خزانه: جستجو و فیلتر نوع گردش.
- مشتریان: تب نوع + طلب/بدهی. مطالبات: بازهٔ سن + حداقل مبلغ + مرتب‌سازی.

== ۳) صفحهٔ چک (بازنویسی کامل) ==
- دو بخش جدا: دریافتی و صادره.
- محاسبهٔ دقیق «روز مانده» با تبدیل تاریخ شمسی به میلادی.
- وضعیت‌ها: در جریان / نزدیک سررسید (تا ۷ روز) / معوق / پاس‌شده / برگشتی،
  با هشدار بالای صفحه برای معوق و نزدیک.
- خلاصهٔ «دریافتنی در جریان» و «پرداختنی در جریان».
- «پاس شد» با انتخاب حساب مقصد:
    • چک دریافتی → واریز به حساب (+)
    • چک صادره  → برداشت از حساب (−)
    • ثبت تراکنش + سند حسابداری دوطرفه به‌صورت خودکار.
- چک پاس‌نشده «در جریان وصول» است و در مانده حساب لحاظ نمی‌شود؛
  مانده فقط هنگام پاس‌شدن تغییر می‌کند (مطابق منطق حسابداری).
- برگشت‌خوردن چک، ویرایش و حذف.

== حساب‌ها و خزانه‌داری (کامل) ==
- شماره حساب، کارت، شبا، نوع حساب.
- انتقال وجه با فرم کامل مبدأ→مقصد + سند خودکار.
- هر واریز/برداشت/انتقال سند حسابداری دوطرفه می‌سازد.
- مانده لحظه‌ای از گردش + هشدار مغایرت.
- گزارش بانک/صندوق بازه‌ای (treasury/report).
- ویرایش/حذف گردش با بازمحاسبهٔ مانده.

== دفتر کل و سرفصل‌ها (کامل) ==
- ساختار درختی کل/معین/تفصیلی با والد.
- کد خودکار بر اساس نوع (دارایی 1000، بدهی 2000، ...).
- واردسازی کدینگ استاندارد با یک دکمه.
- جلوگیری از حذف سرفصل دارای گردش یا زیرمجموعه.
- نمایش مانده هر سرفصل.

== اسناد حسابداری دوطرفه (کامل) ==
- چند خط بدهکار/بستانکار با افزودن/حذف خط.
- کنترل زنده تراز بودن (دکمه ثبت تا تراز نشود غیرفعال است).
- شماره سند خودکار + وضعیت پیش‌نویس/قطعی.
- ویرایش و حذف کامل سند.

== دستیار: عملیات + گزارش تحلیلی روی بخش‌های جدید ==
- «چک ... به علی»، «چک علی رو پاس کن»، «واریز/برداشت/انتقال».
- «وضعیت چک‌ها چطوره؟» → معوق/نزدیک/در جریان.
- «موجودی حساب‌ها چقدره؟» → تفکیک هر حساب.

API جدید: cheques/:id/pay, cheques/:id/bounce, treasury/transfer,
treasury/report, treasury/movement (PUT/DELETE),
accounting/chart (tree + import-standard), accounting/journal (PUT).
همه در server/index.js و server-standalone.js هماهنگ و تست‌شده.

تست‌ها (سبز): tsc بدون خطا، build موفق، تراز آزمایشی بدهکار=بستانکار،
پاس چک → اثر صحیح روی حساب، گزارش‌های دستیار درست.
