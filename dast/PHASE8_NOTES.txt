دست راست — فاز ۸ (UX داخلی: دیالوگ، منوی لمسی، تقویم شمسی + تراز/سود‌وزیان)
==========================================================================

اجرا: npm install && npm start → http://localhost:8787
یا RUN-WINDOWS-NO-INSTALL.bat | ادمین: admin@dastrast.local / Admin12345

== ۱) پاپ‌آپ‌های داخلی به‌جای پنجره‌های مرورگر ==
ماژول جدید src/ui.tsx:
- confirmDialog / promptDialog / alertDialog: پاپ‌آپ داخلی با UI برند
  (به‌جای confirm/prompt/alert مرورگر). همهٔ تاییدها و ورودی‌ها حالا
  داخل اپ و هماهنگ با تم نمایش داده می‌شوند.
- toast: پیام موفقیت/خطای شناور پایین صفحه.
- DialogHost و ContextMenuHost در main.tsx نصب شده‌اند (سراسری).

== ۲) منوی لمسی (کلیک‌راست / لمس طولانی) + انتخاب گروهی ==
- هوک useLongPress: روی هر ردیف عملیاتی، با کلیک‌راست یا نگه‌داشتن
  انگشت (~۰.۴۵ ثانیه، با ویبره) یک منوی شیشه‌ای باز می‌شود
  (مثل لمس طولانی پیام در پیام‌رسان‌ها): ویرایش / انتخاب چندتایی / حذف.
- حالت انتخاب: دکمهٔ «انتخاب» بالای لیست تراکنش‌ها، انتخاب چندتایی با
  تیک، نوار پایین با «تعداد + جمع مبلغ» و «حذف گروهی».
- پیاده‌سازی نمونه روی «تاریخچه تراکنش‌ها» (الگوی قابل‌گسترش به بقیه).

== ۳) رفع باگ خروج از فیلد هنگام تایپ ==
علت: کامپوننت Form داخل بدنهٔ کامپوننت تعریف شده بود و هر کلید →
remount و از دست رفتن فوکوس. فرم‌های «افزودن حساب» و «سرفصل» به
کامپوننت‌های سطح‌بالا (AccountForm / ChartForm) منتقل شدند. حالا تایپ
روان است.

== ۴) ویرایش/حذف تراکنش‌ها ==
- پاپ‌آپ ویرایش (عنوان/مبلغ/دسته/نوع) + حذف تکی + حذف گروهی.

== ۵) تقویم شمسی برای همهٔ فیلدهای تاریخ ==
- کامپوننت JalaliDatePicker: با کلیک، تقویم شمسی با رنگ‌بندی برند باز
  می‌شود (تبدیل دقیق شمسی↔میلادی، روز امروز و روز انتخابی مشخص،
  دکمهٔ «امروز»). جایگزین ورودی متنی تاریخ در:
  چک (ثبت/ویرایش) و سند حسابداری.

== تراز آزمایشی (کامل) ==
- کد/عنوان/نوع/بدهکار/بستانکار/مانده هر حساب.
- فیلتر بازهٔ زمانی، جمع کل و کنترل توازن، خروجی CSV.

== صورت سود و زیان (کامل) ==
- تفکیک درآمد عملیاتی/غیرعملیاتی، بهای تمام‌شده، سود ناخالص،
  سود/زیان خالص و حاشیهٔ سود + بازهٔ زمانی.
- نمودار ترکیب هزینه‌ها (دونات) و درآمدها (میله افقی).

== دستیار ==
- «وضعیت سود و زیان من چطوره؟» و «تراز آزمایشی چطوره؟» گزارش می‌دهد.
- (از قبل) عملیات و گزارش چک/خزانه/حساب‌ها.

API جدید/به‌روز: trial-balance (days, totals, balanced),
profit-loss (days, تفکیک‌ها، byCat). هماهنگ در هر دو سرور.

تست‌ها (سبز): tsc بدون خطا، build موفق، تراز balanced=true،
سود ناخالص و حاشیه درست، گزارش‌های دستیار درست، GET / = 200.
