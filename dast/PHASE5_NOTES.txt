دست راست — فاز ۵ (UX فرم‌ها + OCR + اصلاح تخصیص + دسته‌بندی + دراپ‌داون‌ها)
=========================================================================

اجرا:
  npm install && npm start    → http://localhost:8787
یا ویندوز بدون نصب: RUN-WINDOWS-NO-INSTALL.bat
ادمین: admin@dastrast.local / Admin12345

۱) الگوی استاندارد UX برای لیست‌ها
   کامپوننت‌های جدید:
   - ListHeader: عنوان + «فیلد جستجو» + دکمهٔ «افزودن» که پاپ‌آپ باز می‌کند
     (فیلدهای ورودی داخل پاپ‌آپ، نه inline).
   اعمال شد روی: حساب‌ها، مشتریان، کارشناسان، دسته‌بندی‌ها، فاکتورها، پروژه‌ها
   (اشخاص از قبل این الگو را داشت). فیلد بالای لیست دیگر «نام» نیست، «جستجو» است.

۵) OCR واقعی رسید/فاکتور
   - ماژول جدید src/ocr.ts: اگر توکن Vision نبود، روی خودِ دستگاه با
     Tesseract.js (fas+eng) متن رسید را می‌خواند و بزرگ‌ترین مبلغ منطقی +
     کلیدواژه‌های «جمع کل/قابل پرداخت» را استخراج می‌کند.
   - Tesseract از CDN در زمان اجرا بارگذاری می‌شود (مرورگر واقعی).
   - جریان: ابتدا تلاش سرور (Vision در صورت توکن) → سپس OCR محلی →
     در نهایت ورود دستی. پیام «در حال خواندن/پردازش رسید» نمایش داده می‌شود.

۶) تخصیص هوشمند شخص + دکمه‌های اصلاح
   - «پنجاه تومن به علی دادم» → به نزدیک‌ترین «علی» تخصیص داده می‌شود و
     سایر علی‌ها (مثل علی محسنی/علی سعادتی) به‌صورت دکمهٔ «اصلاح» زیر پیام
     نمایش داده می‌شوند؛ با یک کلیک تراکنش به فرد درست منتقل می‌شود.
   - دیگر مسیر را با سوال مسدود نمی‌کند (UX سریع‌تر).

۷) دسته‌بندی‌ها
   - رفع باگ: دسته‌بندی‌های پیش‌فرض اپ حالا واقعاً نمایش داده می‌شوند
     (۱۶ دستهٔ استاندارد) — قبلاً به‌خاطر مقداردهی اشتباه seed خالی می‌ماند.
   - برچسب «پیش‌فرض» روی دسته‌های سیستمی + تب‌های همه/پیش‌فرض/سفارشی.
   - ویرایش و حذف برای همهٔ دسته‌ها (حتی پیش‌فرض) کار می‌کند.

۸) فیلدهای انتخابی به دراپ‌داون جستجوشونده تبدیل شدند
   کامپوننت جدید SearchSelect (دراپ‌داون + جستجو + افزودن مقدار جدید):
   - فاکتور: انتخاب مشتری از لیست (جستجو) به‌جای تایپ دستی
   - خزانه‌داری: انتخاب صندوق/حساب از لیست
   - پروژه‌ها: انتخاب مشتری از لیست
   - سند حسابداری: انتخاب حساب بدهکار/بستانکار از سرفصل‌ها

هماهنگی سرور:
   تغییرات منطق دستیار و دسته‌بندی در هر دو server/index.js و
   server-standalone.js اعمال شد.

تست‌های انجام‌شده (سبز):
   tsc بدون خطا، build موفق، GET / = 200،
   categories = ۱۶ در هر دو سرور،
   ابهام «علی» → تخصیص به علی محسنی + دکمهٔ علی سعادتی،
   parse-receipt محلی amount=0 → سپس OCR سمت کلاینت فعال می‌شود.
