دست راست — فاز ۶ (رفع باگ نام + تکمیل اشخاص و مطالبات)
======================================================

اجرا:
  npm install && npm start    → http://localhost:8787
یا ویندوز بدون نصب: RUN-WINDOWS-NO-INSTALL.bat
ادمین: admin@dastrast.local / Admin12345

== رفع باگ تخصیص نام در دستیار ==
مشکل: «۵۰ تومن به امیر راوی دادم» اشتباهاً به شخص «امیر» تخصیص می‌شد.
علت: تابع findPersonCandidates نام کوتاه را داخل نام جستجو هم تطبیق می‌داد.
رفع: حالا اگر کاربر نام کامل (چندکلمه‌ای) بگوید، فقط با تطابق دقیق یا
«نام شخص شامل عبارت» مطابقت می‌شود؛ تطبیق با نام کوچک فقط وقتی است که
کاربر یک کلمه گفته باشد. نتیجه:
  - «۳۰ تومن به امیر دادم»      → شخص «امیر»
  - «۵۰ تومن به امیر راوی دادم» → شخص جدید «امیر راوی» (نه امیر)
  - «۲۰ تومن به امیر دادم»      → امیر + دکمهٔ اصلاح «امیر راوی»

== اشخاص / بدهکار-بستانکار (کامل شد) ==
- پروفایل کامل: نام، موبایل، کد ملی، آدرس، نوع شخص، برچسب‌ها، یادداشت
  (فرم پاپ‌آپ کامل برای افزودن و ویرایش)
- صفحهٔ جزئیات شخص: کارت مانده، دفتر حساب، اطلاعات تماس و برچسب‌ها
- محدودیت حذف: اگر شخص سند داشته باشد، حذف رد می‌شود و فقط با تایید
  «حذف اجباری» انجام می‌شود.
- ادغام اشخاص تکراری: انتخاب «ادغام» روی یک شخص و کلیک روی شخص مقصد →
  همهٔ تراکنش‌ها/چک‌ها/پروژه‌های او منتقل و شخص تکراری حذف می‌شود.
- خروجی Excel/CSV از دفتر حساب شخص (با BOM فارسی).
- ارسال یادآوری بدهی: لینک واتساپ آمادهٔ ارسال با متن خودکار؛ اگر موبایل
  نبود، متن کپی می‌شود.
سرور: GET/POST/PUT/DELETE /api/persons (+ فیلدهای جدید)،
      POST /api/persons/merge، GET /api/persons/:id/reminder،
      حذف محافظت‌شده با ?force=1

== مشتریان و مطالبات (کامل شد) ==
- تفکیک نوع: تب‌های مشتری / تامین‌کننده / کارشناس / همه (بر اساس فیلد kind)
- صفحهٔ مطالبات بازنویسی شد با «گزارش سن مطالبات (Aging)»:
  نمودار چهار بازه (۰-۳۰، ۳۱-۶۰، ۶۱-۹۰، بیش از ۹۰ روز) قابل کلیک برای فیلتر.
- فیلتر پیشرفته: جستجو، حداقل مبلغ، مرتب‌سازی بر اساس مبلغ یا قدمت.
- یادآوری دریافت وجه برای هر بدهکار (لینک واتساپ).
سرور: GET /api/receivables/aging، GET /api/persons?kind=customer

== OCR رسید (وضعیت) ==
OCR با Tesseract.js روی خودِ دستگاه کار می‌کند. موتور در «اولین استفاده»
از CDN دانلود و کش می‌شود (بار اول اینترنت لازم است)؛ بعد آفلاین کار می‌کند.
اگر توکن Vision در پنل ادمین تنظیم شده باشد، آن اولویت دارد.

== هماهنگی ==
همهٔ تغییرات منطق (نام، اشخاص، مطالبات) در هر دو server/index.js و
server-standalone.js اعمال و تست شد.

تست‌ها (سبز): tsc بدون خطا، build موفق، GET / = 200،
امیر/امیر راوی دو شخص جدا، aging=۲۰۰، merge=۲۰۰، reminder با لینک واتساپ.
