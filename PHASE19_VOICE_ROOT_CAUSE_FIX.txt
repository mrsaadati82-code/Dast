فاز ۱۹ — رفع ریشه‌ای باگ «متنی تشخیص داده نشد» در ضبط ویس
============================================================

علت اصلی (تفاوت نسخهٔ جدید با نسخهٔ قدیمی dastrust-old):
----------------------------------------------------------
در نسخهٔ قدیمی، پاپ‌آپ ضبط فقط یک انیمیشن سادهٔ CSS داشت و تنها مصرف‌کنندهٔ
میکروفون، خودِ webkitSpeechRecognition بود → تشخیص گفتار سالم کار می‌کرد.

در نسخهٔ جدید، کامپوننت VoiceWave (موج صوتی گرافیکی) اضافه شد که داخل خودش
navigator.mediaDevices.getUserMedia({audio:true}) را صدا می‌زد تا موج واقعی
رسم کند. یعنی «همزمان» دو مصرف‌کننده میکروفون را می‌خواستند:
  1) SpeechRecognition (سرویس تشخیص گفتار مرورگر/گوگل)
  2) getUserMedia (برای AnalyserNode موج)

روی اندروید (Chrome/WebView) میکروفون عملاً در اختیار یک مصرف‌کننده است؛
وقتی getUserMedia میکروفون را می‌گرفت، session تشخیص گفتار صدای خالی/قطع‌شده
دریافت می‌کرد → onresult هیچ‌وقت متن نمی‌داد → «متنی تشخیص داده نشد».
(روی بعضی دسکتاپ‌ها این تداخل پیش نمی‌آید، برای همین فقط روی موبایل خراب بود.)

راه‌حل (src/App.tsx):
----------------------
1) VoiceWave دیگر اصلاً getUserMedia صدا نمی‌زند؛ AudioContext و AnalyserNode
   حذف شدند. موج به‌صورت شبیه‌سازی‌شده رسم می‌شود و یک prop جدید به نام
   boost گرفته که سطح انرژی موج را کنترل می‌کند.
2) در ChatScreen، رویدادهای خود SpeechRecognition
   (onsoundstart / onspeechstart / onresult) تابع pulseWave() را صدا می‌زنند
   که boost را برای ~۹۰۰ms بالا می‌برد → موج هنگام صحبت واقعی پرانرژی می‌شود
   بدون این‌که میکروفون دوباره باز شود.
3) onresult مقاوم‌تر شد: نتایج final و interim جداگانه جمع می‌شوند
   (r.isFinal) تا متن پایدار با متن موقت قاطی/پاک نشود؛ متن خالی هم دیگر
   متن قبلی را overwrite نمی‌کند.
4) sendVoice علاوه بر state از voiceTextRef هم می‌خواند (محافظ در برابر
   race بین آخرین onresult و کلیک ارسال).
5) cleanup تایمر boost روی unmount اضافه شد.

تست‌ها:
- npx tsc --noEmit → بدون خطا
- npm run build → موفق (dist/index.html ~465KB / gzip ~139KB)
- node --check server/index.js و server-standalone.js → OK
- اجرای سرور و curl روی / و /api/health → 200

نکته: Web Speech API روی Chrome اندروید به اینترنت نیاز دارد و روی
Firefox پشتیبانی نمی‌شود؛ این محدودیت مرورگر است، نه باگ اپ.

به‌روزرسانی — رفع تکرار متن روی اندروید (interim تجمعی)
----------------------------------------------------------
گزارش کاربر: گفت «پنجاه تومن هزینه کردم بابت خرید خونه» ولی متن استخراج‌شده
تکرار پشت‌سرهم بود: «۵۰۵۰ تومان۵۰ تومان هزینه...».

علت: باگ شناخته‌شدهٔ Chrome اندروید — نتایج interim تجمعی برمی‌گردند؛ هر
نتیجهٔ جدید کل جمله را از ابتدا تکرار می‌کند. چسباندن سادهٔ همهٔ نتایج
(حلقهٔ += روی e.results) باعث تکرار می‌شد.

راه‌حل (onresult): ادغام هم‌پوشان تکه‌ها —
  * اگر تکهٔ جدید با آخرین تکه شروع شود (تجمعیِ کامل‌تر) → جایگزین آخرین تکه
  * اگر آخرین تکه با تکهٔ جدید شروع شود (تکراریِ کوتاه‌تر) → نادیده
  * در غیر این صورت (سگمنت مستقل، رفتار دسکتاپ) → به انتها اضافه

تست منطق ادغام با node:
  * نتایج تجمعی اندروید → «۵۰ تومان هزینه کردم بابت خرید خونه» ✓
  * سگمنت‌های مستقل دسکتاپ → درست ✓
  * حالت ترکیبی final+interim → درست ✓
tsc بدون خطا، build موفق.
