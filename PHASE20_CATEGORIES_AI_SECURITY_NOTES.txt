فاز ۲۰ — تکمیل دسته‌بندی‌ها + دستیار هوشمند (آموزش/قوانین) + امنیت
======================================================================
(هر دو سرور server/index.js و server-standalone.js کاملاً همگام شدند)

۱) دسته‌بندی‌ها (بخش ۱۵ سند کمبودها)
--------------------------------------
بک‌اند:
- db.categoryMeta: متادیتای هر کاربر { [name]: { color, icon, parent } }
- GET /api/categories/meta → متادیتا + تعداد استفادهٔ هر دسته در تراکنش‌ها
- POST /api/categories: پذیرش color/icon/parent (والد باید موجود و غیر از خودش باشد)
- PUT /api/categories/:name — تغییر نام با «همگام‌سازی کامل»:
    * تراکنش‌های همان کاربر → نام جدید
    * سرفصل حسابداری هم‌نام + خطوط اسناد (accountTitle) → نام جدید
      (تراز آزمایشی تراز می‌ماند؛ تست شد: debit=credit بعد از تغییر نام)
    * اصلاحات یادگرفته‌شدهٔ دستیار (corrections.category) → نام جدید
    * متادیتا منتقل و والدِ زیر‌دسته‌ها اصلاح می‌شود
    * جلوگیری از نام تکراری (409)
- DELETE: حذف اجباری → تراکنش‌های دسته به «سایر» منتقل می‌شوند تا گزارش‌ها
  و اسناد بی‌صاحب نشوند؛ والد زیر‌دسته‌ها خالی می‌شود.
فرانت (CategoriesScreen):
- پالت ۱۰ رنگ + ۱۶ آیکون + انتخاب دستهٔ والد در فرم افزودن/ویرایش
- نمایش درختی: زیر‌دسته‌ها تورفته زیر والد + رنگ/آیکون/تعداد تراکنش
- هشدار شفاف هنگام تغییر نام (همگام‌سازی خودکار) و حذف اجباری

۲) دستیار هوشمند — آموزش و قوانین AI (بخش ۱۶ سند کمبودها)
------------------------------------------------------------
بک‌اند:
- POST /api/assistant/dry-run: پردازش جمله روی «کپی» دیتابیس → هیچ ثبتی
  انجام نمی‌شود (تست شد: تعداد تراکنش قبل/بعد برابر)
- POST /api/ai/rules/test: تست قانون قبل از ذخیره (matched + نتیجهٔ شبیه‌سازی)
- قوانین: weight (۱-۱۰۰)، enabled، scope (user/global)
    * فقط ادمین می‌تواند global بسازد/ویرایش/حذف کند
    * موتور محلی هر دو سرور: قوانین فعالِ کاربر+عمومی، مرتب بر اساس وزن
- PUT/DELETE /api/ai/rules/:id و PUT/DELETE /api/training/:id
- GET /api/assistant/corrections + DELETE …/:id (تاریخچهٔ آموخته‌ها)
- GET /api/ai/rules/export و POST /api/ai/rules/import (JSON، بدون تکراری)
فرانت:
- TrainingScreen سه تب شد: آموزش (با ویرایش/حذف) | چت‌بات تست (dry-run
  زنده با توضیح تشخیص) | تاریخچهٔ اصلاحات (مشاهده/حذف)
- AdvancedAIScreen: وزن، دامنهٔ عمومی (فقط برای ادمین)، تست قبل از ذخیره،
  فعال/غیرفعال‌سازی، ویرایش/حذف، Export/Import فایل JSON،
  برچسب «عمومی/وزن/غیرفعال» روی هر قانون

۳) امنیت (بخش ۱۹ سند کمبودها)
--------------------------------
- کلید مشترک: data/secret.key تصادفی (۳۲ بایت) — هر دو سرور همان را
  می‌خوانند → توکن ورود بین npm start و standalone سازگار است.
  (قبلاً دو secret هاردکد متفاوت بود → توکن بین سرورها نامعتبر می‌شد)
- رمزنگاری توکن AI با AES-256-GCM (enc:v1:iv:tag:data) +
  سازگاری عقب‌رو با مقدار متنی قدیمی؛ callAi در هر دو سرور decrypt می‌کند.
- سیاست رمز: ۸+ کاراکتر، حرف+عدد + اعتبارسنجی فرمت ایمیل در ثبت‌نام
- قفل ورود: ۵ تلاش ناموفق → ۱۵ دقیقه قفل (هر دو سرور)
- POST /api/auth/change-password (با رمز فعلی)
- نقش‌ها: PUT /api/admin/users/:id/role + محافظت «حداقل یک ادمین»
- Audit Trail: recordAudit در رویدادهای مهم؛ GET /api/admin/audit (ادمین)
- بکاپ خودکار در writeDb: هر ۶ ساعت یک نسخه در data/backups؛ ۱۴ نسخهٔ
  آخر نگه‌داری؛ GET/POST /api/admin/backups برای مشاهده/بکاپ فوری
- اعتبارسنجی: badAmount روی تراکنش/چک/خزانه (متناهی، نامنفی، سقف 1e15)؛
  نوع تراکنش فقط income/expense؛ PUT تراکنش نمی‌تواند id/userId/createdAt
  را بازنویسی کند.
- رفع باگ: ثبت‌نام در standalone حالا اولین کاربر را admin می‌کند (مثل express)
فرانت:
- صفحهٔ جدید «امنیت حساب» (همهٔ کاربران): تغییر رمز + فهرست حفاظت‌ها
- AdminPanel چهار تب شد: تنظیمات | کاربران (تغییر نقش) | لاگ تغییرات | بکاپ

تست‌های انجام‌شده (curl روی هر دو سرور)
------------------------------------------
- رمز ضعیف رد، رمز قوی ثبت، قفل بعد از ۵ تلاش (هر دو سرور) ✓
- تغییر نام دسته → tx + chart + journal همگام؛ تراز آزمایشی balanced ✓
- dry-run بدون ثبت واقعی ✓ — تست قانون matched + دسته درست ✓
- قانون global ادمین: کاربر عادی می‌بیند/استفاده می‌کند ولی حذف نمی‌تواند ✓
- export/import قوانین ✓ — تاریخچهٔ اصلاحات ✓
- توکن AI رمزنگاری‌شده در db.json (enc:v1:...) و decrypt صحیح در callAi ✓
- تغییر رمز (غلط→خطا، درست→ورود با رمز جدید) ✓
- audit و backups در پنل ادمین، دسترسی کاربر عادی به audit رد شد ✓
- تنزل آخرین ادمین رد شد؛ ارتقا/تنزل عادی کار می‌کند ✓
- tsc بدون خطا، build موفق، node --check هر دو سرور ✓
