================================================================================
دست راست (Dast Rast) — فهرست کامل بخش‌ها، گزینه‌ها، امکانات و عملیات‌ها
نسخه: فاز ۲۰ — برای تحلیل هوش مصنوعی (شناسایی نیازها و کمبودهای هر بخش)
================================================================================

درباره محصول:
دستیار مالی و حسابداری فارسی با رابط موبایلی (PWA) + بک‌اند محلی Node.js.
دو سرور هم‌ارز: server/index.js (Express، با npm start) و server-standalone.js
(بدون وابستگی، node server-standalone.js). دیتابیس: JSON محلی (data/db.json).
تک‌منبع حقیقت مالی: اسناد حسابداری دوطرفه (journalEntries) — همهٔ عملیات
(تراکنش، چک، خزانه، فاکتور، تسویه) سند خودکار می‌سازند.

کاربران هدف: شخص، کسب‌وکار کوچک، تیم خدماتی/پروژه‌ای، حسابدار.
احراز هویت: JWT (HS256) + نقش‌های admin/user.

- توکن ۳۰ روزه؛ خروج (پاک‌کردن توکن از localStorage)
API: POST /api/auth/register | POST /api/auth/login |
     POST /api/auth/change-password | GET /api/me

- دسته‌بندی، شخص، پروژه، بانک، نوع تراکنش
امکانات دیگر:
- مقایسه دوره‌ای (ماه/فصل/سال) با درصد تغییر نسبت به دورهٔ قبل
- هشدارهای مالی هوشمند: هزینه>درآمد، بزرگ‌ترین بدهکار/بستانکار، چک نزدیک سررسید
- تراکنش‌های اخیر + فرم ثبت سریع تراکنش + دسترسی سریع به دستیار
API: GET /api/analytics/summary|filters|compare|alerts |
     GET/PUT /api/dashboard/prefs

بخش ۳ — دستیار هوشمند (چت)
================================================================================
ورودی‌ها:
- متن فارسی آزاد (تایپ)، ضبط صدا (Web Speech API با fa-IR)، عکس رسید
  (OCR محلی Tesseract.js یا مدل Vision در صورت اتصال توکن AI)
- دکمه‌های پیشنهادی آماده (مثال‌های یک‌لمسی)
عملیات قابل فهم با زبان طبیعی (موتور محلی، بدون نیاز به اینترنت/توکن):
1. ثبت هزینه/درآمد: «۵۰ تومن دادم بابت تاکسی» (تشخیص مبلغ، دسته، بانک، عنوان)
2. ثبت طلب/بدهی شخص: «به علی ۲۰۰ تومن قرض دادم» / «از رضا ۱ میلیون قرض گرفتم»
3. ساخت خودکار شخص جدید + پیشنهاد اشخاص هم‌نام (دکمه‌های اصلاح تخصیص)
4. ثبت چک دریافتی/صادره با تاریخ شمسی: «چک ۵ میلیونی از احمد گرفتم برای ۱۵ مهر»
5. پاس‌کردن چک: «چک احمد رو پاس کن» (اثر روی حساب + سند خودکار)
6. ویرایش آخرین تراکنش/چک با دستور طبیعی: «مبلغ آخرین تراکنش رو ۳۰۰ کن»
7. حذف با تایید دو مرحله‌ای (دکمهٔ «بله، حذف کن» / «انصراف»)
8. بازگردانی (Undo/Restore) آخرین عملیات با دکمهٔ «بازگردان»
9. تسویه کامل حساب شخص: «حساب علی رو تسویه کن»
10. واریز/برداشت/انتقال خزانه: «انتقال ۲ میلیون از صندوق به بانک ملت»
11. ساخت پروژه با مشتری: «پروژه سایت برای مشتری اکبری ۵۰ میلیون»
12. تسویه/پرداخت کارشناس
13. چند عملیات در یک پیام (جداکنندهٔ ؛ یا ; یا «و» یا خط جدید)
14. پرسش تحلیلی: «چقدر خرج رستوران کردم؟» (پاسخ محلی یا مدل خارجی)
امکانات تعاملی روی پیام‌ها:
- دکمه‌های «طلب از او / بدهی به او / هزینه معمولی» (reclassify)
- دکمهٔ «اصلاح دسته‌بندی» → یادگیری ماندگار (corrections)
- منوی لمسی روی پیام: کپی، حذف پیام، پاک‌کردن کل گفتگو
- رفع ابهام مرحله‌ای برای اشخاص هم‌نام با دکمه‌های انتخاب
ضبط ویس:
- پاپ‌آپ اختصاصی با موج گرافیکی (تغذیه از رویدادهای SpeechRecognition —
  بدون باز کردن میکروفون دوم)، توقف خودکار با ۳ ثانیه سکوت،
  دکمه‌های انصراف/توقف/ضبط مجدد/ارسال، پنل «جزئیات فنی ضبط» برای عیب‌یابی،
  ادغام هم‌پوشان نتایج تجمعی (رفع باگ تکرار متن در Chrome اندروید)
API: POST /api/assistant/command | confirm-delete | undo | correction |
     dry-run | GET/DELETE /api/assistant/corrections |
     POST /api/transactions/:id/reclassify |
     POST /api/ai/parse-transaction | parse-receipt | ask

================================================================================

- تست قانون قبل از ذخیره (متن نمونه → matched + نتیجهٔ شبیه‌سازی)
- Export/Import قوانین+آموزش‌ها (JSON، بدون تکراری)

ضمیمه — نرمال‌سازی متن فارسی (برای درک بهتر هوش مصنوعی)
================================================================================
موتور محلی این نرمال‌سازی‌ها را روی ورودی کاربر انجام می‌دهد. هنگام تحلیل،
این قواعد را معادل بدانید:

۱) یکسان‌سازی حروف عربی/فارسی:
   ي → ی        ك → ک
   مثال: «علي» = «علی»، «بانك» = «بانک»

۲) تبدیل ارقام فارسی/عربی به انگلیسی:
   ۰۱۲۳۴۵۶۷۸۹ و ٠١٢٣٤٥٦٧٨٩ → 0-9
   حذف جداکنندهٔ هزار: ٬ و ,

۳) اعداد حرفی فارسی → عدد (با پشتیبانی «و»):
   یه/یک=۱، دو=۲، سه=۳، چار/چهار=۴، پنج=۵، شیش/شش=۶، هفت=۷، هشت=۸، نه=۹،
   ده=۱۰ ... نوزده=۱۹، بیست=۲۰، سی=۳۰، چهل=۴۰، پنجاه=۵۰ ... نود=۹۰،
   صد/یکصد=۱۰۰، دویست=۲۰۰ ... نهصد=۹۰۰، «نیم»=۰.۵
   مثال: «دو میلیون و پانصد» = 2,500,000

۴) واحدهای پول (تفسیر مبلغ):
   - N میلیون/ملیون/م = N×1,000,000
   - N هزار / N کا / Nk = N×1,000
   - N تومن/تومان: اگر N<1000 → N×1,000 (عرف محاوره: «۵۰ تومن»=۵۰هزار تومان)
   - N ریال = N÷10 تومان
   - عدد تنها <1000 کنار فعل پولی (دادم/گرفتم/خریدم/خرج/...) → ×1,000

۵) املاهای محاوره‌ای معادل:
   تومن=تومان، ملیون=میلیون، پونصد=پانصد، پونزده=پانزده، چار=چهار،
   شیش=شش، یه=یک، اخرین=آخرین، اره=آره

۶) واژه‌های ایست نام شخص (هرگز جزو نام نیستند، از انتهای نام بریده می‌شوند):
   پول، مبلغ، قرض، چک، طلب، بدهی، بدهکار، بستانکار، دادم، گرفتم، پرداخت،
   دریافت، بده، واریز، برداشت، تومن، تومان، میلیون، ملیون، هزار، ریال،
   بابت، برای، رو، را، که، از، به، با، کردم، شد، شدم + هر عدد
   پیشوندهای حذف‌شونده: آقا، خانم، جناب
   مثال: «به امیر راوی دادم» → نام = «امیر راوی» (نه «امیر راوی دادم»)

۷) الگوهای تشخیص جهت مالی:
   طلب (receivable): طلب دارم، طلبکارم، بستانکارم، قرض دادم، پول دادم به،
                      چک دریافتنی، از ... طلب
   بدهی (payable): بدهکارم، بدهی، قرض گرفتم، باید بدم، باید پرداخت،
                    چک پرداختنی، به ... بدهکار
   درآمد: واریز، حقوق، درآمد، دریافت، گرفتم، فروش، سود، پورسانت، دستمزد
   هزینه: دادم، پرداخت، برداشت، خریدم، خرج، هزینه، کرایه، قسط، قبض، شارژ
   قاعدهٔ ترکیب با شخص:
   «به/برای X + دادم/پرداخت» → طلب از X | «از X + گرفتم» → بدهی به X

۸) کلیدواژه‌های دسته‌بندی خودکار:
   حمل و نقل: تاکسی، اسنپ، تپسی، مترو، اتوبوس، بنزین، سوخت، پارکینگ، کرایه
   خوراکی و سوپرمارکت: سوپر، مارکت، نان، نونوایی، میوه، قصابی، مرغ، برنج، لبنیات
   رستوران و کافه: رستوران، کافه، ناهار، شام، صبحانه، فست فود، پیتزا، کباب، قهوه
   حقوق و درآمد: حقوق، واریز، درآمد، فروش، سود، پورسانت، دستمزد
   اقساط و بدهی: قسط، وام، بدهی، اقساط، تسهیلات
   مسکن و اجاره: اجاره، رهن، خانه، منزل، دفتر
   قبوض و خدمات: قبض، برق، آب، گاز، تلفن، اینترنت، شارژ
   درمان و سلامت: دکتر، دارو، درمان، بیمارستان، آزمایش، دندان
   پوشاک: لباس، کفش، پوشاک | آموزش: مدرسه، دانشگاه، کتاب، دوره، کلاس
   سفر: سفر، هتل، بلیط، پرواز، قطار
   تفریح و اشتراک: تفریح، سینما، بازی، اشتراک، نتفلیکس، فیلیمو، اسپاتیفای
   سرمایه‌گذاری: سرمایه، بورس، طلا، دلار، صندوق، کریپتو، ارز

۹) بانک‌های قابل تشخیص در متن: ملی، ملت، پاسارگاد، سامان، صادرات

۱۰) تاریخ شمسی:
   «N ماه‌نام» (مثل «۱۵ مهر») → سال جاری/MM/DD
   yyyy/mm/dd یا yyyy-mm-dd → نرمال به yyyy/mm/dd
   تبدیل دقیق شمسی→میلادی برای محاسبهٔ روز مانده

۱۱) جداکننده‌های چند-عملیات در یک پیام:
   ؛ یا ; یا خط جدید — یا «و» وقتی دو الگوی «عدد+فعل پولی» وجود دارد
