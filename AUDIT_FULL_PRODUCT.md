# بخش سوم: عملیات‌های مالی

## ۳.۱ ثبت هزینه
- **منطق:** detectTransaction → type=expense → tx + سند: بدهکار «دسته‌هزینه» / بستانکار «حساب نقد (bank یا صندوق)»
- **فیلدها:** title, amount, category, bank, date(شمسی), method, note, personId?, projectId?
- **قوانین:** amount متناهی، ۰<amount≤1e15؛ بدون مبلغ → noop با سوال (slot-filling)
- **محاسبات:** ماندهٔ حساب نقد ↓، هزینهٔ دوره ↑، بودجهٔ دسته spent ↑

## ۳.۲ ثبت درآمد
- **منطق:** type=income → سند: بدهکار «نقد» / بستانکار «دستهٔ درآمد»
- **قوانین:** کلمات درآمدی (واریز/حقوق/فروختم...) بدون فعل هزینه → income

## ۳.۳ ثبت طلب (receivable)
- **منطق:** «به X دادم/قرض دادم» → accountingSide=receivable → سند: بدهکار «حساب‌های دریافتنی» / بستانکار «درآمد» — **نقد جابه‌جا نمی‌شود**
- **قوانین:** هشدار سقف اعتبار لحظه‌ای اگر ماندهٔ جدید > creditLimit؛ دکمه‌های reclassify (طلب/بدهی/معمولی)

## ۳.۴ ثبت بدهی (payable)
- **منطق:** «از X گرفتم/قرض گرفتم/بدهکارم» → سند: بدهکار «هزینه» / بستانکار «حساب‌های پرداختنی»

## ۳.۵ تسویه و بازپرداخت (settlement)
- **منطق:** settlementDelta: منفی=وصول طلب (نقد+، دریافتنی−)، مثبت=پرداخت بدهی (پرداختنی−، نقد−)
- **قوانین حیاتی:** گارد جهت — «بدهیم به X رو پس دادم» وقتی X به ما بدهکار است → پیام راهنما نه ثبت غلط؛ سقف وصول=|مانده| (۹۰۰ از طلب ۵۰۰ → فقط ۵۰۰)؛ تشخیص سوم‌شخص «X بدهیش رو پس داد»=وصول
- **محاسبات:** مانده شخص → صفر/کاهش؛ صندوق ± دقیق

## ۳.۶ انتقال بین حساب‌ها
- **منطق:** سند: بدهکار «مقصد» / بستانکار «مبدأ» — درآمد/هزینه نیست، در cash-flow خنثی
- **قوانین:** مبدأ≠مقصد (400)؛ canonicalAccountTitle: «صندوق اصلی/نقد/کش»→«صندوق»

## ۳.۷ ثبت چک (دریافتی/صادره)
- **فیلدها:** amount, dueDate(شمسی/نسبی), type, personName, bank, serial
- **قوانین:** بدون مبلغ → سوال؛ تا پاس‌نشدن **در مانده اثر ندارد**
- **محاسبات:** daysUntil با تبدیل دقیق شمسی→میلادی (کبیسه)

## ۳.۸ پاس‌کردن چک
- **منطق:** payChequeEffect: گردش خزانه + تراکنش نمایشی + یک سند (دریافتی: نقد بد/وصول چک بس — صادره: پرداخت چک بد/نقد بس)
- **قوانین:** چک پاس‌شده دوباره پاس نمی‌شود (409)

## ۳.۹ جریمهٔ دیرکرد
- **فرمول:** `fee = balance × monthlyPct/100 × (overdueDays/30)` که `overdueDays = debtAge − graceDays`
- **قوانین:** فقط با policy فعال؛ ثبت = tx طلب جدید + سند + lastLateFeeAt

## ۳.۱۰ فاکتور و دریافت وجه
- **محاسبات:** subtotal=Σ(qty×price)؛ discount=Σردیفی+کلی+تخفیف اختصاصی مشتری (subtotal×discountPct٪)؛ tax=(subtotal−discount)×taxRate٪؛ amount=taxable+tax؛ balance=amount−paid
- **دریافت:** tx درآمد + اثر حساب + سند + status=paid اگر paid≥amount

## ۳.۱۱ بودجه/هدف/هشدار (عملیات نرم)
- بودجه: upsert per-category؛ spent=monthSpent (تراکنش‌های expense ماه شمسی جاری)
- هدف: saved+=deposit؛ done اگر saved≥target
- این‌ها **سند حسابداری نمی‌سازند** (متادیتای رفتاری‌اند) — صحیح است

|---|---|
